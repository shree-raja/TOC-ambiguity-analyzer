// 01_WHAT_IS_AMBIGUITY?

In Theory of Computation, a Context-Free Grammar (CFG) is ambiguous if there exists at least one string in its language that can be generated by two or more distinct Parse Trees (or equivalently, two or more unique leftmost derivations). This indicates structural overlap in the production rules.

// 02_THE_ROOT_CAUSE

Ambiguity typically arises when a grammar fails to define Precedence (which operators bind tighter) and Associativity (whether identical operators group left-to-right or right-to-left). Without these constraints, a parser cannot deterministically resolve how to build the syntax tree.

// 03_RESOLUTION_STRATEGY

We solve this by refactoring into an Unambiguous Grammar using Stratification. By introducing hierarchical Non-Terminals (e.g., splitting a generic Expr rule into Expr, Term, and Factor), we force higher-precedence operations to be evaluated deeper in the parse tree.

// 04_SUPPORTED_SYNTAX

Non-terminals: any word in ALL_CAPS or single uppercase letter (e.g., S, Expr, Stmt). Terminals: lowercase words or symbols (e.g., a, +, id, while). Epsilon: use eps or epsilon for empty productions. Multi-char terminals must be space-separated in target string.
